# Architecture Decision Records (ADR)

Registros breves de decisiones técnicas relevantes del proyecto. Formato inspirado en [MADR](https://adr.github.io/madr/) en versión ligera.

| ID | Título | Estado |
|----|--------|--------|
| [0001](0001-eleccion-de-stack.md) | Elección del stack y arquitectura base | Accepted |
| [0002](0002-observabilidad-tres-pilares.md) | Observabilidad como tres pilares correlacionados | Accepted |

## Convenciones

- Numeración correlativa (`NNNN-titulo-en-kebab-case.md`).
- Cada ADR documenta una sola decisión.
- Estados habituales: **Proposed**, **Accepted**, **Deprecated**, **Superseded by ADR-XXXX**.
- Lo que cambia tras la entrega vive en un ADR nuevo, no se reescribe el anterior.
